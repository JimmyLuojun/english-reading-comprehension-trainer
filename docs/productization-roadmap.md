## 1. 阶段路线

### 阶段 0：现状巩固（当前）

**做什么：**
- 继续完善 Web UI、AI 分析、卡片、复习等核心功能
- 保持 `pytest` ≥ 90% 覆盖率、关键模块 100%
- 新增功能仍按 AGENTS.md "无测试不合入" 原则

**不做什么：**
- 不重构成多前端框架
- 不引入鉴权 / 多用户
- 不投入打包

### 阶段 1：自用稳定化（短期）

**做什么：**

1. **本地服务安全**
   - FastAPI 显式绑定 `127.0.0.1`，禁止 `0.0.0.0` 暴露公网
   - 启动时生成随机 token，所有请求需 header 校验
   - 跨设备访问通过 Tailscale / 局域网，不直接开端口
2. **数据目录抽象**
   - 把 `data/` 路径变成可配置（环境变量或配置文件）
   - 当前自用仍可指向项目内 `data/`
   - 目的：为以后迁移到用户 App 数据目录留口子
3. **响应式阅读页**
   - 阅读、复习页适配窄屏 / 平板
   - 优先满足"手机访问桌面 FastAPI"的体验
4. **可选：拆 `fastapi_app.py`**
   - 当单文件影响维护时再拆 `routers/ templates/ static/`
   - 不为拆而拆

**触发"进入阶段 2"的条件：**
- 用户明确表态"要给别人用了"
- 或：发现"启动 FastAPI → 浏览器访问" 工作流在他人手上跑不动

### 阶段 2：桌面 App 打包（条件触发，未启动）

**做什么（届时再细化）：**

- **PoC 优先**：先做最小 Tauri + FastAPI sidecar Demo，验证：
  - 包体（预期 150–300MB+）
  - 冷启动时间（预期 2–7 秒）
  - sidecar 启动失败的恢复策略
  - macOS / Windows 签名公证流程是否可接受
- **PoC 通过后**：才投入完整重构
- 架构方向：

  ```text
  Tauri Desktop Shell
    -> bundled Python FastAPI sidecar (PyInstaller)
      -> existing services: importers / cards / review / ai / profile
        -> SQLite + assets directory (in user data dir)
    -> WebView renders responsive UI
  ```

**届时必须先做的前置工作：**
- 数据目录迁移（项目内 `data/` → 用户 App 数据目录），含首启迁移逻辑和备份
- AI key 改为系统安全存储或加密本地配置
- 启动期健康检查、splash、sidecar 崩溃恢复

### 阶段 3：分发能力（阶段 2 之后）

- 自动更新（Tauri updater，注意 PyInstaller 整包替换体积）
- 错误上报 / 崩溃日志
- 安装包代码签名（Apple Developer / Windows EV / Authenticode）
- 文档：用户手册、安装指南、隐私政策

### 阶段 4：移动端（远期，可能不做）

**重要：这一阶段实质是"加云端后端"，不是"再包一层壳"。**

- 现状 FastAPI 强耦合本地 SQLite + 文件系统 + spaCy，无法直接进 iOS / Android 沙箱
- 现实路径：
  - **A**（最低成本）：响应式 Web + Tailscale，手机浏览器访问桌面 FastAPI → 自用已够
  - **B**：PWA + Capacitor 壳，但需要先把后端做成可远程访问 + 鉴权 → 等同于"加一层云"
  - **C**：完全云端化 → 重新评估架构，本路线图作废
- **第一版移动端只做**：阅读、复习、播放发音、查看卡片
- **不做**：EPUB 导入、AI 分析、文件管理 → 留在桌面端

## 2. 关键风险

### R1：PyInstaller + Python sidecar 的真实成本

- **包体**：FastAPI + spaCy + en_core_web_sm + Pillow 起步 150MB，含模型 300MB+
- **启动慢**：PyInstaller onefile 冷启动 2–5 秒 + spaCy 加载 1–2 秒
- **不能交叉编译**：需要 macOS / Windows / Linux 各自构建机或 GitHub Actions 矩阵
- **杀软误报**：Windows Defender / 国内杀软常误判 PyInstaller 产物 → 需要代码签名证书
- **更新策略**：整包替换每次更新 100MB+ 下载

**缓解**：阶段 2 必须先做 PoC 验证可接受度，PoC 不通过则放弃 Tauri 路线，转 PWA + 云后端

### R2：spaCy 是包体和启动时间的最大单点

- `en_core_web_sm` 约 50MB
- 已有 `english_lemmatizer.py` fallback，可在打包时换轻量替代
- 或把分句 / 词形还原从启动期常驻改成按需子进程

### R3：本地 FastAPI 的浏览器 CSRF 风险

- 必须 `127.0.0.1` 绑定 + 随机端口 + token
- 否则用户浏览器其他标签页可发请求到本地服务

### R4：数据目录迁移的破坏性

- 从项目内 `data/` 迁到 App 数据目录是一次性操作
- 必须有迁移前自动备份 + 回滚路径 + 旧路径覆盖配置项
- 阶段 1 末期就要把"数据路径可配置"做好，阶段 2 才有可能平稳迁移

### R5：阶段 4 是断崖

- "移动 = 加云端"，不是"延伸"
- 不要在阶段 1–3 做任何"未来移动端需要"的妥协（比如提前拆云端接口、引入鉴权框架）
- 等真的要做移动时再评估

## 3. 暂不投入清单（重要）

以下事项在 **明确触发条件出现前不做**，避免被推着做过早工作：

- [ ] Tauri 桌面壳
- [ ] PyInstaller 打包
- [ ] 代码签名 / Apple Developer 账号 / Windows EV 证书
- [ ] 自动更新机制
- [ ] 多用户鉴权 / OAuth / 注册登录
- [ ] 云端后端 / 多设备同步服务
- [ ] PWA / Service Worker / 离线缓存
- [ ] Capacitor / 原生移动端
- [ ] 遥测 / 崩溃上报 SDK
- [ ] 计费 / 订阅 / License 校验
- [ ] App Store / Google Play 上架准备

**触发条件汇总：**
- 用户明示"要给别人用了" → 启动阶段 1 末期 + 阶段 2 PoC
- 用户明示"要做手机原生 App" → 重新评估架构，本文档需大改
- 自用过程中浏览器访问 FastAPI 的体验崩塌 → 才考虑桌面打包
