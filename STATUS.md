# Project Status

Updated: 2026-06-20

## Current State

- The project is in the self-use stabilization phase: local FastAPI Web UI, SQLite storage, TXT/EPUB/PDF imports, AI analysis/cache, SM-2 review, cards, and EPUB media support are active.
- `app/web/fastapi_app.py` has been split into a thin app factory plus `routers/`, `queries/`, `views/`, and shared web helpers while preserving `create_app(db_factory=None)`.
- `app/web/services/` now holds workflow services for book deletion, TXT/EPUB/PDF import outcomes, and AI analysis/card-cache coordination.
- PDF import Phase 2B is implemented through `app/importers/pdf_importer.py`: normalized selectable text plus coarse vector/image figure-region preservation and math/code non-prose region preservation as `book_assets` and `chapter_blocks(kind='figure')`; it does not introduce a PDF viewer or OCR.
- Cards 页 Word Cards 表格现在提供直接 Delete 操作，复用现有词卡软删除接口；Source/词条链接继续用于回到来源阅读位置或已保存的 Word Analysis。
- Word Cards 的 `Occ.` 现在表示不同来源位置数量，由 `word_card_sources` 表记录；旧词卡迁移时按 `first_sentence_id` 回填并重算，Cards 页提供 Sources / Find Occurrences / Set primary。
- EPUB import now treats obvious `Part ...` divider pages as non-counted sections and strips word-based `Chapter One:` style prefixes from displayed chapter labels.
- PDF import now detects page-leading `Part ...` and `Chapter ...` headings before falling back to fixed 10-page virtual chapters; detected title lines are removed from trainable prose while body text remains.
- DeepSeek API routing now defaults ordinary analysis to `deepseek-v4-flash`, sentence AI analysis to `deepseek-v4-pro`, and exposes a reader `Reanalyze with Pro` action backed by `TRAINER_PRO_MODEL`.
- Reader nested Word Analysis now passes analysis-panel block context as transient `context_text`, so words selected from explanations are analyzed in the explanation context while source links still use the original reading sentence anchor.
- Word Analysis now passes the learner's saved Word Card Note into immutable prompt `word_analysis.v4.md` as `learner_note`; AI returns a separate `learner_note_check` block that evaluates the note without overwriting My notes or the original meaning analysis.
- Reader Write translation now repositions the expanded translation editor around the target sentence after it opens, keeping the sentence visible instead of covering it.
- Reader `Save only` for sentence translations now keeps saved translations as a separate `translated` dotted-underline state without adding the sentence to Review; `Check translation` still saves analysis and reactivates the sentence card for Review; `Delete translation` clears the translation and archives the sentence card because the review reason is gone.
- Reader sentence analysis now shows a deterministic `Similar past mistake` section when the current Check translation diagnosis shares confident error codes with active translated Review sentence cards; it reuses `error_types`, `sentence_card_errors`, and saved `diagnosis_evidence` instead of adding a second mistake taxonomy.
- Root `Makefile` now provides shared verification entry points (`make test`, `make test-web`, `make ruff-web`) that always use `english-reading-trainer/.venv/bin/python`, keeping Claude/Codex/local runs out of miniconda base or system Python.
- Reader Word Analysis now falls back to the previous saved valid analysis when a fresh POST analysis fails validation or the LLM call fails, and marks the fallback stale with a warning instead of returning 502 when a saved analysis exists. AI cache writes now allow a later valid response to replace an earlier invalid row for the same key.
- Sentence Analysis now surfaces editable `Your translation` and `Takeaway` fields in the reader panel, with Takeaway stored in the existing `sentence_cards.user_note` column. Cards shows a Takeaway column, and sentence Text links deep-link back to `/read/{book_id}?chapter={idx}&sentence_id={id}&panel=analysis`.
- Dashboard metric cards now act as navigation shortcuts: Books opens the book list, Sentence cards and Word cards jump to anchored Cards sections, and Due now opens Review. The top Books nav is a client-side resume link when `reader:last-book-id` exists, while Library opens the full book list.
- Cards 页 Sentence Cards 的 Translation 和 Takeaway 列现在参照 Word Cards Notes 使用内容加铅笔按钮的内联编辑；Translation 复用现有 `/mark/sentence/{sentence_id}/translation` 保存语义，Takeaway 复用 `sentence_cards.user_note`，Text 列完整显示原句。
- Reader 的 AI Analysis 桌面布局采用方案 A：从进入阅读页起永久预留右侧 520px 分析栏，正文在剩余左区居中，面板开关不改变正文位置；关闭时右侧保留 Analysis 折叠标签；低于 1180px 宽度时回退为全屏分析面板，避免压窄正文。
- AI Analysis 递归式学习优化采用最小方案：句子 prompt 新版本只增加 `blocking_point` 和 `takeaway_suggestion`，词/词组/固定搭配 prompt 新版本只增加 `role_in_sentence`；Reader panel 按“整句答案 → 卡点 → 结构 → 诊断 → 回到整句 → Takeaway”重排，并提供 Accept suggestion 预填现有 Takeaway，不新增表或第二套 Review。
- Error taxonomy now includes a fourth `inference` layer with `I01` hidden-relation inference failure and `I02` implicature/stance inference failure. Sentence analysis prompts are bumped to v3; word analysis remains v5.
- Web UI now has a global 米黄 / sepia eye-comfort theme toggle in the shared nav. The preference persists in `localStorage`, is applied before the CSS block to avoid a bright flash, and leaves dark/night mode plus font-size/line-height controls deferred.
- The default Web UI visual language now uses a softer editorial variable layer: teal accent, three text tones, system-serif display headings, unified radii, lighter surfaces, and soft card/table shadows. Sepia mirrors the new variables and semantic reading highlights keep their original colors.
- Visual refresh follow-ups are implemented: `.primary` buttons are filled teal with white text, Dashboard metric numbers use tabular figures, and the shared page head includes an encoded inline SVG favicon.
- UI consistency and density polish is implemented: sepia now uses a warmer white-paper palette, page headers render through a shared `_page_header` helper, Import/Profile use a narrower page width, tables read flatter with lighter hover and muted Delete actions, Review buttons are more compact, and Import forms no longer stretch controls full-width.
- Top-level Books now resumes the last-read book by client-side rewriting `#nav-books` from `reader:last-book-id` and `reader:progress:book:{id}`; Library remains the stable `/books` directory entry. The old toolbar Continue reading buttons are removed, and reader body paragraphs remain at 20px / 1.8 without changing the global UI font baseline.
- Reader/Review UX fixes are implemented: AI analysis rendering no longer closes an active translation editor, edited translations keep existing analysis reachable while marking it stale, Review distinguishes sentence Translation from Takeaway, word/phrase/collocation highlights use distinct colors, analysis-panel word marking refreshes body highlights, and `Alt+S` selects the current sentence (`Alt+T` opens translation).
- Reader sentence translation updates now preserve the existing sentence `ai_analysis_id` as a stale analysis reference instead of removing the panel entry. Staleness is detected from both prompt version and the `ai_cache.content_hash` for the current sentence plus user translation; stale sentence errors are still cleared until a fresh Check translation rewrites them.
- Reader click handling now distinguishes selections on the clicked sentence from stale selections elsewhere, so clicking a different sentence with an existing saved analysis loads that sentence's AI Analysis panel even if a prior sentence selection is still active.
- Reader sentence analysis now uses immutable sentence prompt v4 files that require concrete local modifier dependencies when they exist, while optional Logic markers and Anaphora remain empty unless the sentence actually contains them. The Reader panel hides empty optional structure subsections instead of rendering dash-only blocks.
- Reader now supports sentence structure practice Phase A: whole-sentence toolbar selection can open `Write structure`, save a learner structure attempt into `sentence_cards.user_structure`, and run sentence analysis prompt v5. Optional `structure_feedback` stays in `ai_cache.response_json` with `missed_or_wrong[].error_code` constrained to the structure subset of existing error codes; it does not write structure practice errors into `sentence_card_errors`.
- Reader translation/structure editors now stay open after `Save only`, support debounce auto-save while typing, use `Close` as the manual exit action, and keep the toolbar open while selecting/copying source text. When a whole different sentence is selected while an editor is open, the editor first queues any dirty save against the old sentence id, then refreshes the floating editor for the newly selected sentence so stale translations/structures do not appear or save under the wrong sentence. Toolbar save status has reserved height and the toolbar repositions on dynamic size changes so `Auto saved` feedback does not push the floating editor over the source sentence. The structure editor opens with an editable `主干/从句/修饰成分/指代逻辑` template. The bare template (labels only, no real content) is treated as empty by `structureAttemptHasContent`, so it is never saved or sent for AI analysis. The side `我的结构 / 我的翻译` analysis-panel editors mirror the same behaviour: template prefill, debounce auto-save, and a page-hide/panel-close flush.
- Reader Reanalyze and Reanalyze with Pro now send `force_refresh=1`, skip cache reuse, and allow a new valid LLM response to replace the existing valid cache row for the same content/prompt/model key. Invalid force-refresh responses still do not overwrite valid cached analysis.
- Reader sentence analyses now preserve the translation and structure attempt used for that specific analysis on `ai_cache.input_translation` / `ai_cache.input_structure`. The AI Analysis panel shows read-only "Initial translation analyzed" / "Initial structure analyzed" snapshots only when those inputs differ from the current editable translation/structure. Structure snapshots use compact local line-level change summaries, while translation snapshots use local token/phrase-level summaries so previews show the changed phrase rather than the whole sentence. `is_stale` remains a refresh signal rather than a snapshot-display trigger.
- Reader AI Analysis panel now opens/restores sentence or word analyses at the top instead of restoring the previous internal bottom scroll. Its title/copy/meta tools are available at the top, then auto-collapse into a narrow right-edge three-dot handle while scrolling down; hovering the three dots, right-edge hover, or keyboard focus expands them again. The explicit nested-analysis `Back to previous analysis` path still restores the prior panel scroll position.
- `docs/design.md` is now a compact architecture map and index instead of a continuously growing implementation log.
- Web source files and mirrored tests now align for all non-`__init__.py` files under `app/`; router tests verify route registration, while query/view/helper tests cover stable contracts.
- `python -m ruff check app/web` is part of the Web change verification contract.
